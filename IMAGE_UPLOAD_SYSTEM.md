# Система загрузки изображений

## 📸 Описание

Реализована полноценная система загрузки изображений, которая дополняет существующую возможность ввода URL. Пользователи могут как вводить ссылки на изображения, так и загружать файлы с компьютера.

## 🏗️ Архитектура

### Серверная часть

#### Контроллер загрузки (`Upload.controller.js`)

- **Multer** для обработки multipart/form-data
- Валидация типов файлов (JPEG, PNG, GIF, WebP)
- Ограничение размера файлов (5MB)
- Уникальные имена файлов с timestamp
- Создание папки uploads/images автоматически

#### Роуты (`upload.routes.js`)

- `POST /api/upload/image` - загрузка изображения
- `DELETE /api/upload/image/:filename` - удаление изображения
- Требует авторизации (verifyAccessToken)

#### Статическая раздача файлов

- `/uploads/*` - доступ к загруженным файлам
- Настроена в app.js

### Клиентская часть

#### Компонент ImageUpload

```typescript
interface ImageUploadProps {
  currentImageUrl?: string;
  onImageChange: (imageUrl: string) => void;
  onImageClear: () => void;
  placeholder?: string;
  acceptTypes?: string;
  maxSizeMB?: number;
}
```

**Функциональность:**

- Ввод URL изображений
- Загрузка файлов с drag & drop
- Предварительный просмотр
- Валидация размера и типа файлов
- Интеграция с модальными окнами уведомлений

## 🎯 Использование

### В формах администратора

```typescript
import { ImageUpload } from "@/shared";

// Заменить обычный input на:
<ImageUpload
  currentImageUrl={formInput.img}
  onImageChange={(url) => setFormInput((prev) => ({ ...prev, img: url }))}
  onImageClear={() => setFormInput((prev) => ({ ...prev, img: "" }))}
  placeholder="Введите URL изображения или загрузите файл"
/>;
```

### Обновленные формы

✅ **AdminNewsForm** - загрузка изображений для новостей
✅ **AdminProductForm** - простое редактирование изображения продукта  
✅ **AdminManageProductForm** - полная форма создания продуктов

## 🔒 Безопасность

### Валидация файлов

- **Типы**: только изображения (JPEG, JPG, PNG, GIF, WebP)
- **Размер**: максимум 5MB
- **MIME-type**: проверка реального типа файла

### Авторизация

- Все операции с файлами требуют авторизации
- Проверка JWT токена перед загрузкой

### Уникальные имена

- Timestamp + случайное число
- Формат: `image-{timestamp}-{random}.{ext}`

## 📁 Хранение файлов

### Структура папок

```
server/src/uploads/

    ├── image-1703123457890-987654321.png
```

### URL доступа

- Загруженный файл: `/uploads/images/filename.ext`
- Полный URL: `http://localhost:3000/uploads/images/filename.ext`

## 🎨 UX Сценарии

### Загрузка файла

1. Пользователь нажимает "Загрузить"
2. Выбирает файл в диалоге
3. Происходит валидация
4. Файл загружается на сервер
5. URL автоматически подставляется в поле
6. Показывается предварительный просмотр

### Ввод URL

1. Пользователь вводит URL в текстовое поле
2. Автоматически показывается предварительный просмотр
3. При ошибке загрузки показывается уведомление

## 🚀 Преимущества

### Для пользователей

- **Удобство**: можно и вводить URL, и загружать файлы
- **Предварительный просмотр**: сразу видно результат
- **Обратная связь**: четкие уведомления об ошибках

### Для системы

- **Надежность**: файлы хранятся локально
- **Производительность**: нет зависимости от внешних ресурсов
- **Контроль**: полный контроль над загруженными файлами

## 📊 База данных

### Схема хранения

Столбцы `img` в таблицах продолжают хранить строки:

- **URL изображения**: `https://example.com/image.jpg`
- **Локальный файл**: `/uploads/images/image-123456789.jpg`

### Без изменений в БД

- Существующие данные остаются без изменений
- Новая функциональность работает с текущей схемой
- Обратная совместимость с URL изображений

## 🔄 Workflow

### Создание контента с изображением

```
1. Пользователь открывает форму создания
   ↓
2. Заполняет поля + выбирает способ добавления изображения:
   ├── Ввод URL → сохраняется как есть
   └── Загрузка файла → файл → сервер → URL генерируется
   ↓
3. Отправка формы
   ↓
4. В БД сохраняется строка с URL (локальным или внешним)
   ↓
5. При отображении: <img src={imageUrl} />
```
